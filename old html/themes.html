Create a Theme

The defining features of a theme can be found in the files that compose it. HTML provides the basic structure and content, while CSS provides the visuals. The CSS is where we define our theme.

CSS

The CSS allows us to keep consistency through the website without rewriting code over and over again. Make sure to define thematic colors and structures first, then apply them to your HTML.

Mixins and Colors

For this style guide, we defined mixins and specific colors for our pages that gave it a theme that we were trying to achieve.

.mixinGradientLight{
    background-image: -webkit-linear-gradient(#F5F5F5 5%,#FFF 40%);
    background-image: -moz-linear-gradient(#FFF 5%,#F5F5F5 40%);
    background-image: -ms-linear-gradient(top,#FFF 5%,#F5F5F5 40%);
}
.mixinGradientLightReverse{
    background-image: -webkit-linear-gradient(#fff 5%,#F5F5F5 40%);

LESS & CSS Files

We use the CSS pre-processor LESS to maintain and extend our CSS. To do this, we can use a variety of LESS compilers (listed below) to turn our LESS into CSS.

  • Crunch! (Windows and Mac)
  • winless.org (Windows only)
  • CodeKit (Mac only)

With LESS, we can use nesting, mixins, and variables to better maintain our CSS and make it more readable.

style-guide.less
.sg-style{
    font-size: 15px;
    margin: 0px;
    .brand-title{
        clear: both;
        font-size: 10px;
        letter-spacing: 0.01em;
    }
}
style-guide.css
.sg-style {
    font-size: 15px;
    margin: 0px;
}
.prettyprint.small-text.notices.back {
    clear: both;
    font-size: 10px;
    letter-spacing: 0.01em;
}

For the CSS, follow these basic guidelines:

  • Follow CSS coding standards when authoring your CSS.
  • Use valid CSS when possible. As an exception, use vendor-specific prefixes to take advantage of CSS3 features.
  • Minimize CSS hacks. The obvious exception is browsers-specific support, usually versions of IE. If possible, separate CSS hacks into separate sections or separate files.
  • All possible HTML elements should be styled by your theme (unless it is a child theme), both in post/page content and in comment content.
